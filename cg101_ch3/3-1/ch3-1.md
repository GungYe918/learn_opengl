# CH3 — Matrices as Transformations (2D first, then 4x4 intro)

## CH3-1. 행렬은 “vector를 다른 vector로 보내는 함수”이다: 2D 선형변환과 GLSL `mat2` 적용

### 1) 본 소제목의 학습 범위

본 소제목은 행렬(matrix)을 “숫자 표”가 아니라 **선형변환(linear transformation)**을 나타내는 **함수적 객체**로 정의하고, 2D에서 회전/스케일을 `mat2`로 표현하여 Vertex Shader에서 직접 적용하는 최소 실습을 제공한다. 본 소제목은 다음 범위만을 포함한다.

* 행렬의 의미: (f(\vec{v}))로서의 변환
* 2D 선형변환의 대표 예: rotation, scaling, shear(개념 언급 수준)
* 행렬 곱의 의미: 변환의 합성(composition)이며 **순서가 본질적으로 중요**함
* OpenGL 실습: `mat2` uniform을 통해 triangle 정점을 변환(translation은 다루지 않음)

본 소제목에서는 translation을 의도적으로 배제한다. 이유는 translation은 “원점을 보존하는 선형변환”이 아니며, 이를 동일한 체계로 다루기 위해서는 homogeneous coordinates(동차좌표)가 필요하고, 이는 다음 소제목에서 3x3 또는 4x4로 정식 도입한다.

---

### 2) Math Booster: 선형변환과 행렬의 정의

#### 2.1 선형변환(linear transformation)의 조건

2D 공간에서 변환 (T)가 선형변환이면 다음 두 조건을 만족한다.

1. 덧셈 보존(additivity)
   [
   T(\vec{a}+\vec{b}) = T(\vec{a}) + T(\vec{b})
   ]

2. 스칼라 배 보존(homogeneity)
   [
   T(c\vec{a}) = c,T(\vec{a})
   ]

이 두 조건은 “원점이 원점으로 간다”는 성질을 함의한다. 즉 선형변환은 translation을 포함하지 않는다.

#### 2.2 행렬은 선형변환의 표현이다

2D에서 모든 선형변환 (T)는 어떤 2x2 행렬 (M)에 의해 다음 형태로 표현된다.

[
T(\vec{v}) = M\vec{v}
]

여기서 (\vec{v})는 column vector로 취급한다. 구현 관점에서는 “입력 vector의 성분에 대해, 고정된 계수 조합을 적용해 새로운 성분을 만든다”는 의미로 충분하다.

---

### 3) 2D 회전과 스케일: 표준 행렬 형태

#### 3.1 rotation(회전)

각도 (\theta)에 대한 2D 회전은 다음 행렬로 정의된다.

[
R(\theta)=
\begin{pmatrix}
\cos\theta & -\sin\theta \
\sin\theta & \cos\theta
\end{pmatrix}
]

이때 (R(\theta)\vec{v})는 (\vec{v})의 길이를 보존하며 방향만 회전시킨다.

#### 3.2 scaling(스케일)

x축, y축 방향 배율이 (s_x, s_y)이면

[
S=
\begin{pmatrix}
s_x & 0 \
0 & s_y
\end{pmatrix}
]

---

### 4) 변환의 합성과 “순서가 중요함”의 의미

두 선형변환 (A, B)를 연속 적용한다는 것은 다음과 같이 서술된다.

* 먼저 (B)를 적용하고, 그 결과에 (A)를 적용:
  [
  \vec{v}' = A(B(\vec{v})) = (AB)\vec{v}
  ]

여기서 핵심은 행렬 곱 (AB)는 일반적으로 (BA)와 같지 않다는 점이다. 즉, **같은 두 변환이라도 적용 순서가 다르면 결과가 달라진다.** 이는 graphics에서 “rotate 후 scale”과 “scale 후 rotate”가 서로 다른 결과를 만드는 현상으로 직접 관찰된다.

본 소제목의 실습에서는 순서 효과를 “정확히 보이게” 하기 위해, (1) rotate만 적용, (2) rotate·scale을 서로 다른 순서로 곱해 적용하는 옵션을 제공한다.

## 5) OpenGL 실습: `mat2` uniform을 Vertex Shader에 적용 (GLFW + GLAD)

### 5.1 파일: `main.cpp`

아래 코드는 CH1의 구조를 재사용하지 않도록, 필요한 요소만 최소 형태로 재구성하였다. CMake는 사용자 측에서 구성한다는 전제이며, 코드는 GLFW + GLAD 환경을 가정한다. GLM은 “행렬을 코드로 다루기 위한 라이브러리”로 사용한다.

```cpp
// src/main.cpp
#include <cstdio>
#include <cmath>

#include <glad/glad.h>
#include <GLFW/glfw3.h>

#include <glm/glm.hpp>
#include <glm/gtc/type_ptr.hpp>


static void framebuffer_size_callback(GLFWwindow* window, int w, int h) {
    (void)window;
    glViewport(0, 0, w, h);
}

static GLuint compileShader(GLenum type, const char* src) {
    GLuint sh = glCreateShader(type);
    glShaderSource(sh, 1, &src, nullptr);
    glCompileShader(sh);

    GLint ok = 0;
    glGetShaderiv(sh, GL_COMPILE_STATUS, &ok);
    if (!ok) {
        char log[1024];
        glGetShaderInfoLog(sh, 1024, nullptr, log);
        std::fprintf(stderr, "Shader compile error:\n%s\n", log);
    }

    return sh;
}

static GLuint makeProgram(const char* vsSrc, const char* fsSrc) {
    GLuint vs = compileShader(GL_VERTEX_SHADER, vsSrc);
    GLuint fs = compileShader(GL_FRAGMENT_SHADER, fsSrc);

    GLuint prog = glCreateProgram();
    glAttachShader(prog, vs);
    glAttachShader(prog, fs);
    glLinkProgram(prog);

    GLint ok = 0;
    glGetProgramiv(prog, GL_LINK_STATUS, &ok);
    if (!ok) {
        char log[1024];
        glGetProgramInfoLog(prog, 1024, nullptr, log);
        std::fprintf(stderr, "Program link error:\n%s\n", log);
    }

    glDeleteShader(vs);
    glDeleteShader(fs);
    return prog;
}


int main() {
    if (!glfwInit()) {
        std::fprintf(stderr, "glfwInit failed!\n");
        exit(1);
    }

    glfwWindowHint(GLFW_CONTEXT_VERSION_MAJOR, 3);
    glfwWindowHint(GLFW_CONTEXT_VERSION_MINOR, 3);
    glfwWindowHint(GLFW_OPENGL_PROFILE, GLFW_OPENGL_CORE_PROFILE);

    GLFWwindow* window = glfwCreateWindow(800, 600, "CG101 CH3-1: mat2 transform", nullptr, nullptr);
    if (!window) {
        std::fprintf(stderr, "glfwCreateWindow failed!\n");
        glfwTerminate();
        exit(1);
    }
    glfwMakeContextCurrent(window);
    glfwSetFramebufferSizeCallback(window, framebuffer_size_callback);

    if (!gladLoadGLLoader((GLADloadproc)glfwGetProcAddress)) {
        std::fprintf(stderr, "gladLoadGLLoader failed!\n");
        glfwDestroyWindow(window);
        glfwTerminate();
        exit(1);
    }

    // ---- Shaders: apply mat2 to positions ----
    const char* vsSrc = R"GLSL(
        #version 330 core
        layout (location = 0) in vec2 aPos;

        uniform mat2 uM; // 2x2 linear transform

        void main() {
            vec2 p = uM * aPos;              // linear transform in 2D
            gl_Position = vec4(p, 0.0, 1.0); // lift to clip-space vec4 (no translation here)
        }
    )GLSL";

    const char* fsSrc = R"GLSL(
        #version 330 core
        out vec4 FragColor;
        void main() {
            FragColor = vec4(0.2, 0.8, 0.9, 1.0);
        }
    )GLSL";

    GLuint program = makeProgram(vsSrc, fsSrc);

    // ---- Triangle vertices (2D) ----
    const float verts[] = {
        -0.5f, -0.5f,
         0.5f, -0.5f,
         0.0f,  0.5f
    };

    GLuint vao = 0, vbo = 0;
    glGenVertexArrays(1, &vao);
    glGenBuffers(1, &vbo);

    glBindVertexArray(vao);
    glBindBuffer(GL_ARRAY_BUFFER, vbo);
    glBufferData(GL_ARRAY_BUFFER, sizeof(verts), verts, GL_STATIC_DRAW);
    glVertexAttribPointer(0, 2, GL_FLOAT, GL_FALSE, 2 * sizeof(float), (void*)0);
    glEnableVertexAttribArray(0);
    glBindVertexArray(0);

     // ---- Build a mat2 transform on CPU (static) ----
    // Option A: pure rotation by 30 degrees
    const float deg = 30.0f;
    const float rad = deg * (3.14159265358979323846f / 180.0f);
    const float c = std::cos(rad);
    const float s = std::sin(rad);

    glm::mat2 R(c, -s,
                s,  c);

    // Option B: scale (non-uniform)
    glm::mat2 S(1.2f, 0.0f,
                0.0f, 0.8f);

    // Compose: order matters
    // - M1 = R * S  : scale then rotate (because v' = (R*S)*v = R*(S*v))
    // - M2 = S * R  : rotate then scale
    glm::mat2 M1 = R * S;
    glm::mat2 M2 = S * R;

    // Choose one to observe difference.
    // Start with M1; later switch to M2 and compare.
    glm::mat2 M = M1;

    glUseProgram(program);
    GLint locM = glGetUniformLocation(program, "uM");
    glUniformMatrix2fv(locM, 1, GL_FALSE, glm::value_ptr(M));

    // ---- Render loop ----
    while (!glfwWindowShouldClose(window)) {
        if (glfwGetKey(window, GLFW_KEY_ESCAPE) == GLFW_PRESS)
            glfwSetWindowShouldClose(window, true);

        glClearColor(0.08f, 0.08f, 0.10f, 1.0f);
        glClear(GL_COLOR_BUFFER_BIT);

        glUseProgram(program);
        glBindVertexArray(vao);
        glDrawArrays(GL_TRIANGLES, 0, 3);

        glfwSwapBuffers(window);
        glfwPollEvents();
    }

    // ---- Cleanup ----
    glDeleteVertexArrays(1, &vao);
    glDeleteBuffers(1, &vbo);
    glDeleteProgram(program);

    glfwDestroyWindow(window);
    glfwTerminate();
    return 0;
}
```

### 5.2 실습에서 “관찰해야 하는 사실”의 명세

본 실습은 다음 사실을 검증하기 위해 구성되었다.

1. Vertex Shader에서 `uM * aPos`는 “정점에 선형변환을 적용한다”는 의미를 가진다.
2. `glm::mat2`로 구성한 (R(\theta)), (S)는 수학 정의(회전/스케일)와 동일한 형태로 코드에 대응된다.
3. (R S \neq S R)이며, `M1`과 `M2`를 바꾸어 적용하면 triangle의 최종 형태가 달라진다.
   이는 “행렬 곱의 비가환성(non-commutativity)”이 변환 합성에서 의미하는 바를 시각적으로 확정한다.

## 6) Common pitfalls & debugging tips

* `uM`을 설정했는데 변화가 없을 때: `glUseProgram(program)` 이후에 `glUniformMatrix2fv`가 호출되었는지 확인한다.
* `locM == -1`인 경우: uniform 이름 오타 또는 shader에서 실제로 사용되지 않아 최적화로 제거된 경우를 의심한다.
* 행렬 성분 배치 혼동: GLM 생성자는 열(column) 단위로 해석되는 경우가 많으므로, 현재 코드처럼 명시적으로 값을 넣고 결과를 관찰하는 방식이 안정적이다. 혼동이 발생하면 “회전이 반대로 되는지”부터 관찰해 부호를 교정한다.
* 삼각형이 화면 밖으로 사라질 때: 스케일이 커져 clip-space를 벗어난 경우가 흔하다. (s_x, s_y)를 1에 가깝게 줄인다.
